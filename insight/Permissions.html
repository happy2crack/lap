Permissions

  • If Android Application needs to perform any actions on User's Android Device, it would require Permission.

  • All the Required Permissions for an App are declared in Android Manifest file. If required permission is missing, it will throw runtime exception.

  • Example of entry in Manifest file: 
    		<uses-permission android:name="android.permission.INTERNET" />
		<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
  • When a User Installs an application from Google Play Store, application's Manifest file is checked for required Permissions and shown to the user.

  • Only if user agrees, Installation proceeds in the Device.

  • Hence, only required permissions should be declared in Manifest and fewer the no. of permissions, more likely it is that user installs app.

  • You can also define custom permissions for components within your application, by using permission attribute in the manifest.

  • For example: Adding a permission to Activity limits the ability of other applications to launch a particular Activity.

  • Adding a permission to Broadcast Receivers control which applications can send Broadcast intents to this particular Receiver.

  • Adding a permission to Content Providers controls read/write operations on Content Providers.

  • Adding a permission to Services controls the ability of other applications to start or bind to a Service.

	<permission
		android:name=”com.custom.ACCESS_SERVICE” 
		android:protectionLevel="signature" 
		android:description=”@string/custom_permission_description”> 
	</permission>


	<service
		android:name=”.DownloadService”
		android:permission=”com.custom.ACCESS_SERVICE”>
	</service>
