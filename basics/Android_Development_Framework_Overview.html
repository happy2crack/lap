Android Development Framework Overview

  • Android applications are normally written in Java and executed by custom VM called Dalvik VM.
  • The Android software stack is, a Linux kernel and a collection of C/C++ libraries exposed through an application framework that provides services for, and management of, the run time and applications.

    • Figure below depicts Android Application framework . The Applications section is where a developer's application sits.



  • Linux 2.6 Kernel: Android is created on top of Linux 2.6 Kernel. Linux kernel core features include Security Model (handling of communication between application and System), Memory Management, Process Management, Network communication, Driver Support etc.
  • Android framework: On the top the Linux 2.6 kernel, the Android framework is developed which consists of Android Run Time and Libraries.
    • Android runtime: The Android runtime is composed of Java core libraries and Dalvik virtual machine (DVM).
      Dalvik is a register based Virtual Machine that's been optimized to ensure that a device can run multiple instances efficiently.
    • Libraries: Android includes various C/C++ core libraries such as:
      Open GL : This is graphics library actually a cross-language, cross-platform API used to produce 2D and 3D computer graphics.
      WebKit: Open source web browser engine provides the functionality to display web content and to simplify page loading.
      SQLite: Open source relational database engine designed to be embedded in devices.
      Media Library: To play and record audio and video.
      Secure Sockets Layer (SSL): These libraries are responsible for Internet security.
  • Application Framework: The application framework provides the classes used to create Android applications. You benefit from these features because you don’t have to build any of the functionality they provide. Some of them are:
    Activity manager: Manages the activity life cycle.
    Telephony manager: Provides access to telephony services as well as to certain subscriber information, such as phone numbers.
    View system: Handles the views and layouts that make up your user interface (UI).
    Location manager: Finds the device’s geographic location.
  • Application Layer: Native and third-party applivations are built on the application layer by means of the same API libraries. The application layer runs within the Android run time, using the classes and services made available from the application framework.
  • From kernel to application, the Android operating system has been developed with proven open source technologies. A developer, can therefore build rich applications that have been fostered in the open source community.
